## 4. 코스 구조

**In / Out 코스는 둘 중 하나만 선택**하는 분기 루트입니다.

```
                    ┌─ S자 ─ 갈림길 ─┐
출발/도착 ──────────┤   (Out 코스)    ├─── 동적 장애물 ─── 출발/도착
                    └─ 회전 교차로 ──┘
                         (In 코스, 옵션)
```

| 구분 | 경로 | 핵심 미션 |
|------|------|-----------|
| Out 코스 | S자 → 좌우 갈림길 | 차선 유지, 표지판 방향 인식 |
| In 코스 (옵션) | 회전 교차로 | 1회 이상 회전 후 탈출 (포기 불가) |
| 공통 | 동적 장애물 → 출발/도착 | 장애물 대응, 빨간불 정지 |

### In 코스 Option 의미 (주최측 답변)

> 회전교차로는 **S자 구간과 갈림길 미션을 스킵하는 숏컷**입니다. 난이도와 페널티가 높게 설정되어 있어, 해당 구간을 주행할지는 **참가팀의 선택**이라는 의미에서 "옵션"입니다.  
> — 조교 곽승헌

---

## 5. 미션별 규정 및 페널티

**공통 규칙**
- 차선 이탈: 2바퀴 이상 탈선 시 **+30초**, 이탈 지점 복귀 후 재주행
- 복귀: **팀장이 컨트롤러/손으로 차량을 들어** 복귀
- 미션 실패: 해당 미션 시작점 재출발 + 시간 페널티

| 구간 | 핵심 규정 | 실패/페널티 |
|------|-----------|-------------|
| 출발 | 준비 5분, 차량 조작 금지, 4바퀴 격자 위, 초록불 인식 후 출발 | 5분 초과 → 주행 실패 / 초록불 미인식 → 미션 실패 |
| S자 | 미션 없는 주행, 저속 권장 | 차선 이탈 +30초 |
| 좌우 갈림길 | 좌/우 표지판 랜덤 배치, 표지판 방향대로 주행 | 차선 이탈 +30초, 실패 시 재출발 |
| 동적 장애물 | ArUco 마커 등장 → 정지 후 사라지면 재출발 | 차선 이탈 +30초, 실패 시 재출발 |
| 도착 | 뒷바퀴 격자 위 → Lab-Time 종료, 빨간불 인식 후 정지 | 빨간불 미인식 +30초 |
| 회전 교차로 | 진입 후 1회 이상 회전 뒤 탈출 | 차선 이탈 시 재출발 +30초 |

### 갈림길 표지판 위치 (주최측 답변)

> 표지판은 **갈림길 사이**에 위치합니다.

### 동적 장애물 동작 방식 (주최측 답변)

> ArUco 마커는 **움직이지 않고**, 올라왔다 **내려가는** 방식입니다.

## 6. 주최측 문답 (확정)

### 1차 문답 — 조교 곽승헌

| # | 질문 | 답변 |
|---|------|------|
| 1 | 작년 해커톤 홍보영상 어디서 볼 수 있나요? | 유튜브 링크는 현재 없음. **파일로만 존재**. 링크 확보 시 재공지 예정 |
| 2 | In코스 회전교차로 Option 의미는? | S자·갈림길을 **스킵하는 숏컷**. 난이도·페널티 높음 → **팀 선택 사항** |
| 3 | 갈림길 표지판 위치는? | **갈림길 사이** |
| 4 | 동적 장애물이 움직이나요? | 마커는 움직이지 않고 **올라왔다 내려감** |

### 2차 문답 — 동적 장애물 ArUco 마커 상세

| # | 질문 | 답변 |
|---|------|------|
| 1 | 마커 높이 (바닥 기준) | **바닥에 거의 붙어 있음**. 좌/우 표지판과 거의 동일한 위치 |
| 2 | 내려오는 속도/시간 | **랜덤하게 움직임**. 정지 인지 후 마커 사라질 때까지 **스탑워치 중지** → 기술만 구현되면 안심 가능 |
| 3 | 마커 실제 크기 | 좌/우 표지판과 비슷하지만 **정사각형** (사진 참고) |
| 4 | 바닥 향 카메라에서 인식 가능? | 마커가 바닥과 거의 붙어 있음. 참가팀 화각은 미확인이나, **정면이 보일 정도면 화각에 충분히 들어올 것** |

### 개발 시사점

- 카메라가 바닥(차선) 위주여도 마커는 **표지판과 비슷한 낮은 높이** → 전방 화각에 포함되면 인식 가능
- 마커 등장 타이밍이 랜덤 → **조기 감지 + 안정적 정지** 로직이 중요
- 정지 인지가 확인되면 타이머가 멈추므로, **인식 정확도**가 속도보다 우선

## 8. 기술 구현 요구사항

| 영역 | 내용 |
|------|------|
| AI 활용 | Claude로 기능 구현 |
| Vision (Rule-based) | OpenCV 기반 |
| Vision (Learning-based) | YOLO26n 추천 (OpenCV만으로도 가능) |
| 통신 | ROS2 (D-Racer Kit) |
| 데이터셋 | 4종 객체 영상 제공 → split 후 학습, 추가 데이터 자유 |

**핵심 구현 과제**
1. 신호등 인식 (초록 출발 / 빨간 정지)
2. 좌·우 표지판 방향 인식 (갈림길)
3. 차선 유지
4. 동적 장애물 (ArUco) 대응
5. 회전 교차로 (In 코스 선택 시)
6. ArUco 마커 인식
